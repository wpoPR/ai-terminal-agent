## üîç Issue Tracking Guidelines

**Severity Levels:**
- **Critical:** Blocks functionality, security risk, data loss
- **High:** Major feature broken, significant performance issue
- **Medium:** Minor feature broken, cosmetic bug
- **Low:** Polish, nice-to-have improvements

**Status Values:**
- **Open:** Not yet addressed
- **In Progress:** Being worked on
- **Fixed:** Resolved and verified
- **Won't Fix:** Decided not to address
- **Duplicate:** Covered by another issue

---

**Instructions:**
1. Add new issues with incremental IDs per severity
2. Update status as work progresses
3. Move fixed issues to "Recently Fixed" section
4. Keep descriptions clear and actionable
5. Include reproduction steps for bugs
6. Document workarounds when available
7. All AIs should check this before starting work
