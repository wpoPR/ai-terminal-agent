## ğŸ’¡ Decision-Making Guidelines

**When to Document:**
- Architectural patterns
- Technology choices
- Security approaches
- Major refactoring
- Breaking changes
- API design

- Minor bug fixes
- Cosmetic changes
- Routine updates
- Obvious choices

**Instructions:**
1. Add new decisions with incremental IDs
2. Keep context brief but informative
3. Always document alternatives considered
4. Update status when decisions change
5. Link related decisions
6. Tag appropriately for searchability
